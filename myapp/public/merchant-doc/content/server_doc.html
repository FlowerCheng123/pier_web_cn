当用户在我们的SDK中支付完成之后，我们将会在我们的后台向商家的后台发送异步通知的请求，告诉商家的服务器，该笔支付已经完成。

异步通知接口

品而平台对商户的支付请求数据处理完成后，会将处理的结果（成功才通知，失败和异常不通知）数据通过服务器主动通知的方式通知给商户的通知地址（该地址在商户接入时，由商户提供）。通知机制为：最多通知8次，通知的时间间隔为：1,2,4,6,8,10,12分钟。直到通知被商户成功接受为止，如果通知8次也没有被商户接受成功则不再发送。通知服务通过http post方式提交给商户服务端，商户服务端通过数据流读写的方式或者通知字符串。请求参数列表：

异步通知的请求参数
字段名 参数名称 是否必须
merchant_id 商户订单号 是
charset 字符集 是
sign_type 签名类型(现在只支持RSA) 是
sign 签名 是
商户接到通知后返回结果
字段名 参数名称 是否必须
ret_code 交易结果代码(其中ret_code为“0000”为成功，品而服务器就会停止通知，否则继续通知直到通知机制完成。) 是
ret_msg 交易结果描述 是 

	//回调中返回成功结果
	{
	   "ret_code":"0000",
	   "ret_msg":"OK"
	}
